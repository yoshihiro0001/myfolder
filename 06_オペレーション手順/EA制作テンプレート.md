# 制作テンプレート（LP/サイト）
- 構成例: ヒーロー → ベネフィット → 実績/証拠 → FAQ → CTA
- FAQ例: リスク/サポート範囲/返金/導入手順/必要スキル
- CTA例: 無料相談予約、デモ申込、限定β参加
- 実績提示: 数字/期間/役割を明記
- 追加要素: 口コミ/第三者評価/安心材料（免責/利用条件）

EA/投資ツール向けのドラフト項目（田中さん情報が入ったら差し替え）
- ベネフィット: 「ドローダウンを抑えた長期運用」など、実測値ベースで書く
- 実績/証拠: バックテスト20年の主要数値（年率/最大DD/勝率/平均利得など）、フォワード実績、使用ロット例
- 運用ルール: ロット計算例、推奨証拠金、想定DD時の行動
- 免責/リスク開示: 資金は顧客管理、相場変動リスク、過去実績は将来を保証しない
- サポート範囲: 導入/設定サポート、Q&A、運用助言の線引き
- 価格: 現行EAは36万＋月3%、新商品は6万＋月1%（β想定）
- 段階的オファー: ①資料＋デモ動画 ②限定β ③一般販売
- CTA: 限定β参加申込、デモ動画視聴、Q&A送付（メール/フォーム）

集客フロー（ドラフト）
- チャネル: DM/メール（紹介・コミュニティ）、限定投稿（招待制SNS/フォーラム）、小規模広告テスト
- 導線: チャネル → 簡易LP（実績サマリ＋デモ動画＋FAQ短版） → 詳細LP（数値・免責・導入手順） → CTA
- CTA: ①デモ動画視聴 ②限定β申込フォーム ③Q&A送信フォーム（通話は任意）
- フォロー: 視聴/申込後にFAQ補足メール、自動返信に免責と次ステップを記載
- 計測: チャネル別クリック/視聴/申込、LP内のFAQ閲覧率、CTAクリック率

会員ページ（利用者向け、集客用とは分離）
- 構成: 実績ダッシュボード（バックテスト/フォワード/現在サマリ）、FAQ、導入手順、免責、アップデート通知
- コンタクト: Q&Aフォームのみ（通話は希望者だけ）、サポート範囲を明記
- 区分: β会員/一般会員/現行EA希望者
- 分離: 集客用LPと別ドメインorサブディレクトリ、認証つきでクローズド

LPのデザインメモ（ワイヤーのヒント）
- ヒーロー: 信頼の根拠（年数、DD、透明性）、価格（同額で兄と摩擦なし）を一行で
- ベネフィット: 「低DD」「長期バックテスト」「設定シンプル」「サポート範囲の明確さ」
- 実績: バックテスト主要指標、フォワード成績、利用環境例、ロット例
- FAQ: リスク/免責、必要証拠金、ロット設定、サポート範囲、返金有無、想定DD時の行動
- CTA: 「限定βに申し込む」「デモ動画を見る」「Q&Aを送る」

追記メモ（今回の要件）
- 最低安全入金額: 100万円（推奨）、ただし30-50万円から開始も可能（段階的増額推奨）
- 初期費用のハードルを下げる: 段階払い（導入時3万＋導入後3万）、お試し期間（7-14日）の返金/免除ルール
- 価格の棲み分け: 新商品1%、現行3%（希望者のみ）
- 用語: チャネル=届ける場所/手段、CTA=次にしてほしい行動
- ネーミング案: 「Guardian」「DDシールド」「CalmPips」「LowDD Engine」
- キャッチ案: 「低DD×長期バックテストの安心設計」「20年耐えたルールを1%で使う」「設定1時間、資金は自分で守る運用」
- 初期LP演出: 数値カードのフェード/カウントアップ、CTAは軽いホバー。信頼優先で動きは控えめ。
- 初期CTA: 「デモを見る」「限定βに申し込む」「Q&Aを送る」。β枠数の表示で希少性を出す。
- 段階的差し替え: β後にフォワード実績と導入者の声（匿名/数値）を追加、CTAを強める。

追記のヒント: プロジェクトごとに具体的なコピー案や数値を貼り付ける。
