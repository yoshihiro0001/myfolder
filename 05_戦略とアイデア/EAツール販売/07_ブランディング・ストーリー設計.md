# ブランディング・ストーリー設計（最新版 / 核＝信頼ゲーム / 例は空欄）

## この設計のゴール（最初に核を置く）
結論: **数字じゃ見抜けない。最初に“人（開発者/販売者）”を見る。**
EA販売の本質は、**「開発者を信じられるかのゲーム」**。

あなたの役割は「売り込む人」ではなく、**“信頼の判断材料（人の見抜き方）”と“悪い時の手順”を置く人**。
その結果として「信頼したい→信頼できる」が腹落ちした人だけが、更新通知→申込へ進む。

最優先の届け先:\n+**EAをよく知らない「昔の自分」**。\n+“いちばんシンプルな一本道”で判断できる状態にする。

※このファイルは例文なし（空欄運用）。構成と判断材料の設計だけを残す。

## あなたの強み（“信頼ゲーム”に最適化）

### 性格の強みを“価値提供”へ変換（核に寄せる）
- **常に「人は信用できるか？」で見ている**: これ自体がコンテンツになる（判断材料として価値）
- **想像力・本質を見抜く**: 危ない人/案件の“サイン”を言語化できる
- **勝負勘＋堅さ**: 勝率より「破綻しない」に寄せられる
- **無駄なコミュニケーションが嫌い × 自動化が好き**: “揉めない設計”を先に作れる

## ブランドの核（プロミス＝信頼ゲームのルール）

### 世界観（トーン）
- **クール / 静か / 余白 / 観察 / ルール**
- “ドヤ”ではなく **「判定」**。

## 文章ルール（当たり前ドヤ防止 / あなたの性格に固定）
目的: 正しいことを言うほど“上から”に見える事故を防ぐ。\n+特に「当たり前のこと」を“私の思想/姿勢”として語ると、マウント臭が出やすい。

### 禁止パターン（外向け文章では使わない）
- 「〜と思っています / 〜だと思う」（感想・道徳に見える）
- 「徹底する / 〜しないを宣言する」（当たり前の自己PRに見える）
- 「信頼できると思った人だけ / 無理に来なくていい」（逆説の心理テクに見える）
- 「審査 / 選別 / 落とす」（上からの選別に見える）
- 「当たり前の条件を“売り”として語る」（怒り/不信の起点になりやすい）
  - 例: 「負けた月は手数料をいただきません」等

### 置き換え（低温で“仕様/条件/手順”に落とす）
- 「〜と思う」→ **「仕様として固定する」** / **「条件はこう」**
- 「徹底する」→ **「原則」** / **「運用ルール」**
- 「信頼できる人だけ」→ **「判断材料→更新通知（押さない）→申込」**（次の一手だけ置く）
- 「審査」→ **「申込フォーム（同意チェック/事前確認）」**
- 「当たり前の条件」→ **規約/課金まとめに“仕様として1行”**（LP/投稿では売りにしない）

### 主語ルール（ドヤを消す最短）
- 「私」は最小化。\n+  代わりに **チェック項目 / 判定材料 / 仕様 / 条件** を主語にする。

### “低温の締め”テンプレ（LP/固定/投稿共通）
- 次の一手:
  - 判断材料を見る
  - 判断材料の更新通知を受け取る（押さない）

## “ツボ押し”設計（信頼したい→信頼できるを反復）
読者の感情は「信頼したい」。
そこに対して、あなたは「信頼できる材料」を出す。
この反復が購入に繋がる。

### ツボ押しの順番（構造）
1) **人（開発者/販売者）の信頼判断**（核）
2) **悪い時の手順**（DD/停止/再開）
3) **料金（請求条件の仕様）**（曖昧さを潰す）
4) **線引き**（当たり前を淡々と：解約/返金/サポート）
5) **数字・証拠**（最後に添える）

※数字を先に置くと「見抜けない→不信」に戻るので、順番を崩さない。

## 投稿の柱（コンテンツ・ピラー）

### ピラー1: 信頼判断（核）
- 人（開発者/販売者）を信頼できるサイン
- 危ないサイン（強引/押し付け/決めつけ/盛り）
- “不利な事実を言えるか”を最重要に置く

### ピラー2: 悪い時の手順（信頼を補強）
- 損が出た時の行動（止め方/再開）
- 止める条件/再開条件
- 取引量は式（気分で変えない）

### ピラー3: 仕様（境界）/線引き（揉めないための設計）
- 料金は“仕様”として淡々と書く（当たり前を売りにしない）
- 無料EAの罠（紹介報酬→営業→断れず購入）を切る設計
- 線引き（当たり前を淡々と：解約＝次月停止、返金、サポート範囲、禁止事項）

### ピラー4: 数字・証拠（最後に添える）
- “盛れない形”で出す
- 数字は「信頼の証明」ではなく「判断の材料」

### ピラー5: 仕組み化（あなたが消耗しない）
- 情報は会員ページ/判断材料ページに集約
- Q&Aはフォーム化
- ルールで揉めない
