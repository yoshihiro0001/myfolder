# EAツール販売計画（新商品メイン／現行は希望者のみ）

目的
- 新商品EAを直販し、管理手数料1%で収益化。まず限定枠で実績を作り、自信がついてから拡大。
- 現行EA（兄価格36万＋月3%）は希望者のみ案内し、兄と同額で摩擦を避ける。ロイヤリティは合意後に。

商品ライン
- 商品A（新商品・直販）: 初期6万（仮）、管理手数料1%/月、限定β枠から開始
- 商品B（現行EA・兄価格）: 初期36万、管理手数料3%/月、希望者のみ案内（価格は兄と同額）

運用の前提
- 最低安全入金額: 100万円（自身も100万円から開始）を推奨ラインとして明記する
- 運用は顧客資金で自己管理、こちらで預からない

差別化ポイント
- 透明性: バックテスト20年主要指標、現行フォワード実績、DD、ロットルールを開示
- リスク開示: 想定DDと行動、免責、資金は顧客管理
- 導入の簡単さ: 設定サポート1時間以内、サポート範囲を明確化
- 価格の棲み分け: 新商品1%、現行3%で用途/嗜好を分離

段階的販売ステップ
1) 資料＋デモ動画公開（無料）: 実績サマリ、DD、ロット計算例、免責
2) 限定β（商品A・1%）: 枠数限定、期間限定、サポート範囲/返金有無を明記
3) オープン販売（商品A）: βの実績を反映しLP更新。兄へのロイヤリティはβ結果を踏まえ協議。
4) 希望者のみ商品B案内: 価格は兄と同額、ロイヤリティ前提。摩擦回避を明示。

会員ページ/ポータル構想（透明性を出しつつコミュニケーション最小化）
- 機能案: 実績ダッシュボード（バックテスト/フォワード/現在のパフォーマンスサマリ）、FAQ、導入手順、免責、アップデート通知
- コンタクトの線引き: Q&Aフォームのみ（通話は希望者だけ）、サポート範囲を明記
- 会員区分: β会員（商品A1%）/ 一般会員（商品A1%）/ 現行EA希望者（商品B3%）
- 兄配慮: ネット集客に限定し、既存知人・兄の周辺には積極営業しない旨を内部方針として記載
- 役割分離: 集客用LP/デモサイトと会員ページは別ドメインorサブディレクトリで分離し、会員ページはID/PWでクローズド運用

集客フロー（低コミュニケーション）
- チャネル: DM/メール（紹介・コミュニティ）、招待制SNS/フォーラム、小規模広告テスト
- 導線: チャネル → 簡易LP（実績サマリ＋デモ動画＋FAQ短版） → 詳細LP（数値・免責・導入手順） → CTA
- CTA: デモ動画視聴 / 限定β申込フォーム / Q&A送信（通話は希望者のみ）
- フォロー: 視聴/申込直後の自動返信に免責と次ステップ、FAQ補足メール
- 計測: チャネル別クリック/視聴/申込、LP内FAQ閲覧、CTAクリック率

想定顧客像と優先チャネル
- 既存投資コミュニティ/紹介経由の投資経験者（警戒心が低め、紹介が効きやすい）→ DM/メール
- 小口〜中規模の副業/投資層（最低100万を用意できる人）→ 招待制SNS/フォーラムで限定投稿
- 数字とルールを重視する層 → 簡易LPでDD・ルールを先出し、デモ動画で信頼を稼ぐ
- 広告は小規模テストから（クリック→動画視聴→フォーム入力が取れたチャネルだけ残す）

段階的集客と売上イメージ（ラフ）
- フェーズ0: デモ動画視聴リード 50-100件/月（小規模広告＋DM） → 返信率5-10% → β申込5-10件想定
- フェーズ1: β販売（商品A、6万＋1%）枠10件想定 → 初期売上60万 + 月次1%*運用額（例: 100万×10=1000万 → 月10万）
- フェーズ2: β実績をLP/会員ページに反映 → 一般販売を月10-20件に拡大 → 初期60-120万/月 + 月次1%*（100万×10-20口=1000-2000万 → 月10-20万）
- 商品B（3%）: 希望者のみ案内。価格は兄と同額、ロイヤリティ前提。

用語メモ
- チャネル: 見込み客に情報を届ける場所・手段（DM/メール、限定SNS、広告など）
- CTA: 次にしてほしい具体行動（デモ視聴、β申込、Q&A送信）

LP構成案（完成版の下書き・初期は信頼づくりに集中）
- ファーストビュー（ヒーロー）
  - 1行: 「低DD×20年バックテストのEAを1%で。資金は自分で管理、設定はサポート1時間」
  - 補足: 価格（6万＋月1%）、最低安全入金100万円、限定β枠あり
  - CTA: 「デモを見る」「限定βに申し込む」
  - 動き: 数字カードのフェードイン（最大DD/年率/期間）、CTAボタン軽いホバー
- セクション1 信頼の理由
  - バックテスト20年主要指標（年率/最大DD/勝率/平均利得/期間グラフ）
  - フォワード実績サマリ（期間・成績・環境）
  - 透明性: ロットルール/推奨証拠金/想定DD時の行動を小カードで
  - 動き: 数値カードのカウントアップ、グラフのスライドイン
- セクション2 リスクと免責を先出し
  - 「資金は顧客管理」「相場変動リスク」「過去実績は将来保証しない」
  - 想定DD時の対応フローを図解（例: DD◯%でロット調整/停止基準）
  - 返金有無・サポート範囲を明確に
- セクション3 使い方・導入手順（摩擦を下げる）
  - 3ステップ: 申込 → 環境設定サポート（1時間以内）→ 運用開始
  - 必要なもの: 証拠金100万円〜、MT4/5環境、VPS推奨など
- セクション4 実績詳細（スクロールで深掘り）
  - バックテスト詳細表とグラフ（折れ線/ヒートマップ）
  - フォワードのエクイティカーブ（期間明記）
- セクション5 FAQ
  - 「リスクは？」「必要証拠金は？」「ロット設定は？」「サポートはどこまで？」「返金はある？」「想定DD時にどうする？」
- セクション6 オファー
  - 商品A: 6万＋1%、限定β枠の残数表示
  - 商品B: 36万＋3%、希望者のみ（兄と同額）
  - CTA: 「デモを見る」「βに申し込む」「Q&Aを送る」
- セクション7 免責
  - 簡潔に明記し、詳細はPDF/会員ページで全文

演出/デザインのトーン
- 色: 深い紺＋アクセントで淡いゴールド or エメラルド（信頼＋落ち着き）
- 余白広め、カード型で数値を即視認
- 動きは控えめ（フェード/スライド/カウントアップ）で信頼重視

初期リリースと成長の段階的差し替え
- 初期: デモ動画＋サマリ実績（信頼に足る最低限の数値）、限定βの枠表示を強調
- β後: 実際のフォワード結果・導入者の声（匿名/数値のみ）を追記し、CTA強めに
- 成長後: 会員ページの実績ダッシュボードをサンプル表示し、更新頻度/透明性を訴求

- サポート範囲: 導入/設定/Q&A、運用助言の線引き
- 価格: 商品A=6万＋1%、商品B=36万＋3%（希望者のみ）
- オファー段階: 資料/デモ → 限定β → 一般
